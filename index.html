Paste A + Many B → C Excel（单一受试者版）

A：直接复制粘贴 1–2 行（固定表头，不上传文件）
B：一次可上传 无数个 TXT / JSON / CSV 文件；默认都是 同一个人
系统会自动把 B 分成 pre / post / unknown，并分别从所有 pre / 所有 post 文件中整合字段；如果同一字段不同文件给出不同值，会记录为 conflict，并在 debug 面板里详细列出。

Step 1. 粘贴 A

怎么贴：
直接从 Excel 复制 1–2 行数据 粘贴到下面。
不用贴表头；如果你不小心把表头也一起贴进来，程序会自动尝试跳过。

规则：
- 贴 1 行 → 视为 pre
- 贴 2 行 → 视为同一个人的 pre / post，优先按 timestamp 早晚排序；若无法识别时间，则按粘贴顺序（第一行=pre，第二行=post）
A 表头是固定的；这版不会让你上传 A 文件，也不会处理多个人。

Step 2. 上传 B（可无限多个）

这版的设计就是为了解决你“B 存了很多版本、不知道哪个最新”的问题：
你可以把同一个人的所有 B 全部上传。
系统会：
1. 自动归类为 pre / post / unknown
2. 按字段整合，不是只取一个文件
3. 如果同一字段有冲突，会详细写出来，不会静默覆盖
允许一次上传很多个 B，但默认必须是 同一个人。
如果检测到 B 文件里出现多个不同的 id 或多个不同的 name，程序会报错并停止，防止混入别人数据。

testType 识别顺序：
1. 文件内容中的 testType
2. 文件名中是否含 pre / post
3. 都识别不到 → 归为 unknown（不会参与计算，但会出现在 debug）
